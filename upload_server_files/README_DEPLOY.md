# README_DEPLOY

This folder is auto-generated by `sync-deploy.js`.
Do not edit files here manually; edit source files in the main project.

## Folder Layout

- `frontend/`: production-ready frontend assets and runtime files.
- `backend/`: backend source, routes, models/controllers (inside `src/`), and server entry files.
- `database/`: seed/migrations snapshot for deployment.

## 1) Create ZIP Package

From project root:

```bash
node sync-deploy.js --once
```

Then compress `upload_server_files/` as:

```
upload_server_files.zip
```

## 2) Upload to Server

Upload the zip file to your VPS (example target: `/var/www/dalil-plus`), then:

```bash
cd /var/www/dalil-plus
unzip -o upload_server_files.zip
```

## 3) Install and Run (pnpm + pm2)

### Frontend
```bash
cd /var/www/dalil-plus/upload_server_files/frontend
pnpm install --frozen-lockfile || pnpm install
# If you deploy prebuilt dist only, serve dist with Nginx or static host.
```

### Backend
```bash
cd /var/www/dalil-plus/upload_server_files/backend
pnpm install --frozen-lockfile || pnpm install

# Build if required by your backend setup
pnpm run build || true

# Start or restart via PM2
pm2 start "pnpm start" --name dalil-plus-api || pm2 restart dalil-plus-api
pm2 save
```

## 4) Environment Variables

Use:
- `frontend/.env.production.example`
- `backend/.env.production.example`

Copy and fill real secrets on the server:

```bash
cp backend/.env.production.example backend/.env
cp frontend/.env.production.example frontend/.env
```

Never commit or upload real `.env` values.

## 5) Continuous Sync (Local)

To keep deployment folder always updated while developing:

```bash
node sync-deploy.js
```

It monitors project files and mirrors updates into `upload_server_files/`.
