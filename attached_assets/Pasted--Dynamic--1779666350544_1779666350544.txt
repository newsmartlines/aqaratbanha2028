أريد تحويل المنصة بالكامل إلى نظام Dynamic حقيقي يعتمد على قاعدة بيانات دائمة، بحيث أي شيء أضيفه أو أعدله أو أحذفه من الأدمن أو الموقع يظل محفوظ دائمًا بدون الحاجة لإعادة عمله بعد الـ Deploy أو إعادة تشغيل المشروع.

# مهم جدًا:

لا أريد أي بيانات ثابتة داخل:

* Seeds
* Arrays
* Hardcoded Data
* JSON ثابت داخل الكود

---

# المطلوب:

أي شيء أعمله داخل المنصة يتم حفظه تلقائيًا داخل قاعدة البيانات بشكل دائم.

### أمثلة:

* إضافة مناطق
* تعديل مناطق
* حذف مناطق
* إضافة مدن
* إضافة محافظات
* إضافة أنواع عقارات
* إضافة مميزات
* إضافة خدمات
* تعديل الفلاتر
* الباقات
* الإعدادات
* الـ SEO
* العقارات
* الرسائل
* المستخدمين
* صلاحيات الأدمن
* الصفحة الرئيسية
* الأقسام
* البنرات
* الإحصائيات

كل هذا يكون Dynamic ومربوط بقاعدة البيانات مباشرة.

---

# مهم جدًا:

البيانات الموجودة حاليًا داخل:

* Seeds
* Mock Data
* Static Files

قم بنقلها تلقائيًا إلى قاعدة البيانات الحقيقية بدل بقائها داخل الكود.

### المطلوب:

* عمل Migration / Seeder ذكي
* ينقل كل البيانات الحالية الموجودة بالمشروع إلى Database
* ثم يجعل النظام يعتمد على قاعدة البيانات فقط بعد ذلك

---

# المطلوب تقنيًا:

* Persistent PostgreSQL Database
* ORM احترافي
* Database Migrations
* Dynamic CRUD System
* Admin Dynamic Management System
* Auto Save Changes
* Real Database Storage

---

# مهم جدًا:

اجعل الـ Seeds تستخدم فقط:

* لأول تشغيل للمشروع
* Initial Default Data فقط

بعد ذلك:
أي تعديل يتم من المنصة نفسها يتحفظ دائمًا داخل قاعدة البيانات.

---

# المطلوب أيضًا:

* إنشاء Tables منظمة لكل شيء Dynamic
* عدم الاعتماد على بيانات ثابتة داخل Frontend
* أي صفحة تسحب البيانات من الـ API وقاعدة البيانات مباشرة

---

# مثال مهم:

إذا أضفت:

* منطقة جديدة
* ميزة جديدة
* فلتر جديد

ثم:

* أغلقت المشروع
* عملت Restart
* عملت Deploy
* نقلت المشروع

أريد كل البيانات تظل موجودة كما هي بدون فقدان أي شيء.

---

# Architecture المطلوب:

Frontend
↓
REST API
↓
Backend Logic
↓
Persistent Database

---

# النتيجة:

أريد منصة عقارية احترافية حقيقية مثل الأنظمة العالمية، أي تعديل يتم داخلها يظل محفوظ دائمًا بدون الحاجة لإعادة إدخال البيانات مرة أخرى بعد النشر أو إعادة التشغيل، مع نقل كل البيانات الحالية من الـ Seeds والملفات الثابتة إلى قاعدة البيانات تلقائيًا.
