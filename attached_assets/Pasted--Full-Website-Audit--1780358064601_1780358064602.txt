قم بإجراء فحص شامل (Full Website Audit) لجميع أجزاء الموقع العقاري، سواء الواجهة الأمامية أو لوحة الإدارة أو قاعدة البيانات أو الـ APIs أو نظام الصلاحيات أو الأداء.

المطلوب:

1. اكتشاف وإصلاح جميع الأخطاء البرمجية والمنطقية.

2. مراجعة جميع الصفحات والتأكد من عدم وجود:

   * صفحات معطلة.
   * أخطاء 404.
   * أخطاء Console.
   * أخطاء API.
   * أخطاء Database.
   * أخطاء Routing.
   * أخطاء Forms.
   * أخطاء البحث والفلاتر.

3. مراجعة نظام العقارات بالكامل:

   * إضافة العقار.
   * تعديل العقار.
   * حذف العقار.
   * الموافقة على العقار.
   * رفض العقار.
   * إعادة إرسال العقار بعد التعديل.
   * العقارات المميزة.
   * العقارات المنتهية.
   * ظهور العقارات في الصفحة الرئيسية.

4. مراجعة لوحة الإدارة بالكامل:

   * الجداول.
   * الفلاتر.
   * الإحصائيات.
   * الصلاحيات.
   * الإشعارات.
   * إدارة المستخدمين.
   * إدارة العقارات.

5. مراجعة لوحة المستخدم:

   * إضافة العقار.
   * تعديل العقار.
   * رفع الصور.
   * اختيار الموقع.
   * وسائل الاتصال.
   * الإشعارات.
   * حالة العقار.

6. مراجعة نظام الخرائط:

   * منع أخطاء NaN.
   * التحقق من الإحداثيات.
   * التأكد من ظهور الموقع بشكل صحيح.
   * إصلاح أي مشاكل Leaflet أو Google Maps.

7. مراجعة التصميم بالكامل:

   * توحيد الألوان.
   * توحيد المسافات.
   * تحسين Responsive.
   * تحسين تجربة الموبايل.
   * تحسين الجداول.
   * تحسين النوافذ المنبثقة.
   * تحسين البطاقات.

8. مراجعة الأداء:

   * تحسين سرعة التحميل.
   * Lazy Loading للصور.
   * تقليل الطلبات غير الضرورية.
   * تحسين الكاش.
   * تحسين الاستعلامات.

9. مراجعة SEO:

   * Titles.
   * Meta Tags.
   * Structured Data.
   * Sitemap.
   * Robots.txt.

10. مراجعة الأمان:

* Authentication.
* Authorization.
* Input Validation.
* XSS Protection.
* SQL Injection Protection.
* Rate Limiting.

11. مراجعة جميع الأزرار والروابط:

* التأكد أن كل زر يعمل.
* التأكد أن كل صفحة تفتح بشكل صحيح.
* التأكد أن جميع عمليات CRUD تعمل بدون أخطاء.

12. تنفيذ اختبارات شاملة End-to-End لجميع السيناريوهات المحتملة للمستخدم والإدارة.

13. إصلاح أي مشكلة يتم اكتشافها تلقائياً وعدم الاكتفاء بعرضها.

14. في النهاية قم بإنشاء تقرير تفصيلي يحتوي على:

* الأخطاء المكتشفة.
* الأخطاء التي تم إصلاحها.
* التحسينات المنفذة.
* المشاكل المتبقية.
* اقتراحات التطوير المستقبلية.

مهم جداً:
تصرف كمطور Senior Full Stack + QA Engineer + UI/UX Expert، وافحص الموقع بالكامل صفحة صفحة ووظيفة وظيفة، ثم أصلح أي مشكلة تجدها دون طلب موافقة إضافية.
